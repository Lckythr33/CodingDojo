Clients and Servers

Computers connected to the Web are called takers and givers

  • Clients are typically greedy, data hungry users.
  • Servers are computers that store and distribute the data

So what actually happens?

  1. The browser goes to the DNS server and finds the real address of the server.
  2. The browser sends an HTTP request message to the server asking it to send a copy of the website to the client
  3. If the server is happy it will send a 200ok message which means "Permission granted! Here you go!" and starts sending packets.
  4. The browser assembles the small chunks into a complete website and it displays to you!

So what actually happens?

While the server can process information in many different languages, the files that they serve to the client are always going to be some combination of HTML, CSS and JavaScript!
